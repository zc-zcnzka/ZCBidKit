# 样式设计
新增一级菜单“资源下载”，内容是类似书架那种样式，左侧一个竖版的图片，右侧是标题和简短的描述。
风格要和本项目整体一致，有高级感。
资源内容是从接口读取的，暂时先不做接口，先做静态数据，我们先设计样式。


# 数据管理
@analytics 下增加tab“资源管理”。采用cloudflare D1+R2数据库
以表格形式呈现，包括：图片、标题、标签、介绍、弹窗内容
- 图片存在R2，建立一个OpenBidKit存储桶，D1上存链接
- 图片不是必填项
- 弹窗内容是长文本，一般会放下载链接和下载方式描述等

@client 端资源下载页
- 去掉右上角的文件描述“后续这里会从资源接口读取模板、清单和资料包；当前先用静态数据确认两列书架样式。”，改成一个搜索框和搜索按钮，可以搜索资源
- 数据从Analytics的接口中读取
- 如果图片为空的话，使用现在的随机彩色背景+标题的模式作为默认图片占位
